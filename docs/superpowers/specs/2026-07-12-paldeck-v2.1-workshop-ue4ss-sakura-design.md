# PalDeck v2.1：Workshop、Palworld UE4SS 与樱花重制设计

## 1. 目标

PalDeck v2.1 在现有 Windows 10/11 Steam 客户端版管理器上完成四项升级：

1. 删除当前单色椭圆樱花效果，提供自然写实、日式水彩、极简高级三种可选特效。
2. 将 Okaetsu 的 Palworld 专用 RE-UE4SS 普通用户包原样内置，并提供受信任的显式在线更新检查。
3. 精确检测 Palworld Steam Workshop Mod，并通过 Palworld 自己的 `ActiveModList` 机制直接启停。
4. 新增独立“开源致谢”页面，展示核心集成项目、作者、许可证和 GitHub 链接，并折叠展示完整直接依赖清单。

完成后版本升级为 `2.1.0`，构建新的便携 ZIP 和同名 SHA-256 sidecar，并在公开仓库 `ylty1516/PalDeck` 发布非草稿、非预发布的 `v2.1.0` Release。

## 2. 非目标

- 不切换应用内自更新的受信任仓库；它暂时继续使用 `ylty1516/palworld-mod-manager`。
- 不订阅、退订或删除 Steam Workshop 项目。
- 不修改 Steam 账户订阅状态，不编辑其他游戏的 Workshop 数据。
- 不支持 Xbox/Microsoft Store 或专用服务器管理。
- 不同时加载 Workshop UE4SS 和手动/内置 UE4SS。
- 不将 GitHub 任意 URL 暴露给前端或作为安装输入。

## 3. 已验证事实与来源

### 3.1 Palworld 专用 UE4SS

目标资源为 Okaetsu 的 [`experimental-palworld` Release](https://github.com/Okaetsu/RE-UE4SS/releases/tag/experimental-palworld) 中的 `UE4SS-Palworld.zip`：

- 资产大小：`6,982,837` bytes。
- SHA-256：`7c80b2f4a29baf0f384552c8517e58196e78c8a1b8530637b7179eddae1b54a9`。
- GitHub 资产更新时间：2026-07-09T23:56:19Z。
- 包含 `dwmapi.dll`、`ue4ss/UE4SS.dll`、`ue4ss/UE4SS-settings.ini`、`ue4ss/MemberVariableLayout.ini`、UE4SS Mods 和 `ue4ss/LICENSE`。
- Release 说明它是适配 Palworld 0.4.1.5 后引擎内存布局变化的普通用户包，并警告不能与 Workshop UE4SS 同时安装。
- 项目使用 MIT License；固定源码提交中的版权与许可见 [`LICENSE`](https://github.com/Okaetsu/RE-UE4SS/blob/c2ac246447a8bcd92541070cb474044e7a2bbbe6/LICENSE#L1-L20)。原许可文本要求分发时保留版权和许可声明。

### 3.2 Palworld Workshop

[Palworld Modding Docs 的打包说明](https://pwmodding.wiki/docs/developers/mod-publishing/workshop/packaging)确认：

- Workshop 源目录为 `steamapps/workshop/content/1623730/<WorkshopId>`。
- 每项包含 `Info.json`，记录 Mod 名称、作者、版本、`PackageName`、依赖及 `InstallRule`。
- Lua/UE4SS、Paks、LogicMods 与 PalSchema 会由 Palworld 部署到不同的游戏目录。
- Workshop Lua Mod 不应携带 `enabled.txt`，因为 Palworld 自己的 Mod Manager 负责启停。

[Palworld 官方服务器 Mod 文档](https://docs.palworldgame.com/settings-and-operation/mod/)明确使用 `Mods/PalModSettings.ini` 中重复的 `ActiveModList=<PackageName>` 维护启用列表；本机当前客户端安装也已生成同结构文件。客户端 `UPalModLoaderLibrary` 导出的 `GetActiveModList`、`SetActiveModList`、`ActivateMod` 和 `DeactivateMod` 表明游戏 UI 使用同一套语义。

## 4. 总体架构

新增三个边界清晰的后端组件：

1. `ue4ss_provider`：只负责内置资源、固定元数据、GitHub 检查和已验证下载。
2. `workshop_service`：只负责 Steam 库定位、Workshop 清单读取、`Info.json` 解析、依赖图和 `PalModSettings.ini` 启停。
3. `credits_catalog`：只提供随版本固定的开源项目与直接依赖元数据。

现有 `ue4ss_installer` 继续负责安全解包、目标验证、事务安装和回滚；现有 Mod 列表层把本地清单 Mod 与 Workshop 扫描结果合并成统一的只读视图模型，但两类 Mod 使用不同的启停适配器。

前端仍使用同一 DOM 架构，新增一个 Credits 视图；樱花由新的 Canvas 控制器管理，不引入大型前端框架或外部纹理资源。

## 5. Palworld 专用 UE4SS 设计

### 5.1 第三方资源布局

项目新增独立第三方目录，保存：

- 原始 `UE4SS-Palworld.zip`，不重新压缩、不修改内容。
- `manifest.json`：来源仓库、Release、资产名、资产大小、固定 SHA-256、更新时间和源码参考。
- 上游原始 MIT `LICENSE`。
- PalDeck 的 `NOTICE`：区分 Okaetsu（Palworld Release 维护者）、UE4SS-RE/RE-UE4SS 上游贡献者和许可证版权人 Narknon。

PyInstaller 将该目录作为数据资源嵌入单文件 EXE；便携目录不额外暴露一份可被意外替换的 ZIP。

### 5.2 内置安装

“安装内置 Palworld 专用版”流程：

1. 要求已配置有效 Steam Palworld 路径，且游戏未运行。
2. 读取内置 manifest，以流式 SHA-256 校验内置 ZIP，要求大小和摘要完全匹配。
3. 识别现有手动/Workshop UE4SS；发现 Workshop UE4SS 仍激活时阻止安装并给出关闭入口。
4. 使用既有安全解包策略，额外要求 `dwmapi.dll`、`ue4ss/UE4SS.dll`、`ue4ss/UE4SS-settings.ini` 与 `ue4ss/MemberVariableLayout.ini` 全部存在。
5. 只修改 staging 中的设置；安装目标位于游戏 `Pal/Binaries/Win64`。
6. 先备份冲突文件，再逐文件原子发布；任一步失败则恢复原文件并清理新增目录。
7. 返回来源、摘要、复制文件数、冲突处理和下一步说明。

### 5.3 在线检查与安装

在线操作固定到：

- 仓库：`Okaetsu/RE-UE4SS`
- Tag：`experimental-palworld`
- 资产：`UE4SS-Palworld.zip`

“检查 GitHub 新版本”读取 GitHub Release API 的资产 `size`、`updated_at` 和 `digest`。只有远端 SHA-256 与内置摘要不同时才显示“下载并安装新版”。下载器：

- 只接受 HTTPS GitHub API、`github.com` Release 和 GitHub 资产重定向主机。
- 不接受客户端 URL、仓库名、Tag 或资产名覆盖。
- 设置响应体和落盘大小上限。
- 下载到 F 盘应用 data/cache 下的临时文件。
- 下载后同时校验 GitHub API digest、实际大小和 ZIP 结构，再交给事务安装器。
- 不静默下载、不自动安装；失败时删除临时文件并保留现有安装。

### 5.4 UI

设置页 UE4SS 卡片显示：

- 当前安装状态及布局文件状态。
- 内置资源来源、资产更新时间、短摘要和作者链接。
- “安装内置 Palworld 专用版”“检查 GitHub 新版本”“本地 ZIP”。
- 仅在确认有新版后显示“下载并安装新版”。
- Workshop 冲突、游戏运行中、目标不可写等情况使用结构化错误和明确恢复建议。

## 6. Steam Workshop 检测与启停设计

### 6.1 精确扫描，不遍历整个 Steam 库

1. 从注册表/Steam 配置定位 Steam 根目录。
2. 严格解析 `steamapps/libraryfolders.vdf` 得到少量库路径。
3. 每个库只检查两个固定位置：
   - `steamapps/workshop/appworkshop_1623730.acf`
   - `steamapps/workshop/content/1623730`
4. 只读取 Palworld ACF 列出的数字 Workshop ID 目录；拒绝越界路径、reparse/junction 和非普通文件。
5. 以 ACF 与相关 `Info.json` 的 mtime/size 指纹缓存结果；手动刷新或指纹变化时重扫。

不会遍历其他游戏的 `common` 或 `workshop/content/<其他 AppId>`。

### 6.2 元数据模型

每个 Workshop 项目输出：

- `source=steam_workshop`
- `workshop_id`
- `mod_name`、`package_name`、`author`、`version`
- 安装规则类型与目标
- 依赖 PackageName/Workshop ID
- 源目录、源更新时间、已部署状态
- `enabled`、`global_enabled`、`needs_restart`
- 无效原因或缺失依赖

`Info.json` 字段有类型、长度和字符集限制。缺失/损坏的条目仍可显示 Workshop ID 与错误状态，但不能启用。

### 6.3 与统一 Mod 列表集成

Workshop 项目与本地管理 Mod 一起显示，并带独立“Steam Workshop”徽标。筛选支持名称、作者、类型、PackageName 和 Workshop ID。卡片提供：

- 启用/禁用
- 打开源目录
- 打开对应 Steam Workshop 页面
- 查看依赖与部署状态

不提供删除、移动或退订按钮。

### 6.4 真实启停

启停通过 `Palworld/Mods/PalModSettings.ini` 完成，不移动订阅目录：

- 游戏运行时禁止修改。
- 原子写入并保留 BOM/编码、换行、注释、未知 section 与未知键。
- 启用：去重后添加 `ActiveModList=<PackageName>`；若全局开关为 false，则一并设为 true并明确告知。
- 禁用：只删除对应 PackageName 的 `ActiveModList` 行。
- 写入后重新读取确认，失败则保留原文件。
- 状态明确标注“下次启动游戏生效”。
- 缺少依赖时显示“一并启用依赖”；依赖有环或无效时安全失败。
- 禁用仍被其他已启用项目依赖的项目时二次确认，并列出受影响项目。

Workshop UE4SS 通过 `InstallRule`、PackageName 和已部署标记识别。它与 Win64 手动/内置 UE4SS 形成互斥冲突，不能同时启用。

## 7. 樱花特效重制

当前单色椭圆粒子实现完全删除。新设置由两个独立字段组成：

- `petal_style`：`natural`、`watercolor`、`minimal`
- `petals`：`off`、`low`、`medium`、`high`

旧配置只有 `petals` 时迁移为 `petal_style=natural`，保留原密度。

### 7.1 共享物理层

三种风格共享时间步长受限的粒子系统：

- 三个深度层决定尺寸、速度、透明度和虚化程度。
- 低频噪声改变主风向，短时阵风只影响部分粒子。
- 每粒子有独立旋转轴、翻转相位、漂移和寿命。
- 翻转通过横纵缩放变化表达，不再把下落位置直接当旋转角。
- 页面隐藏时暂停；恢复时不补算长时间差。
- DPR 上限为 2，粒子上限按风格和密度固定。
- `prefers-reduced-motion` 时停止动画并清空 Canvas。

### 7.2 三种渲染器

**自然写实**

- 使用不对称贝塞尔轮廓、粉白渐变、轻微脉络和边缘缺口。
- 每片颜色、透明度与形状有受限随机差异。
- 远景轻微虚化，近景尺寸更大但数量少。

**日式水彩**

- 启动时在离屏 Canvas 预生成有限数量的多层半透明水彩贴图。
- 边缘使用淡色晕染，不使用塑料高光。
- 主要为散落花瓣，低概率出现完整五瓣樱花。
- 运动缓慢、曲线柔和。

**极简高级**

- 粒子数量最少，以白粉色、低饱和、低对比度为主。
- 速度慢、阵风弱，优先从内容区边缘经过。
- 不追踪个人输入，也不做高成本 DOM 碰撞；只依据固定主内容安全区调整生成位置。

### 7.3 设置 UI

设置页显示三个可键盘操作的风格卡片，每张包含轻量 Canvas/静态动效预览及文字说明。选择立即预览，但只有“保存外观”成功后持久化；失败恢复服务端状态。密度控制保留关闭、低、中、高。

外部链接只能来自后端固定 allowlist，通过系统默认浏览器打开；前端不能提交任意 URL。

## 9. API 与错误处理

新增/调整的本机鉴权 API：

- `GET /api/ue4ss/status`
- `POST /api/ue4ss/install-bundled`
- `POST /api/ue4ss/check-upstream`
- `POST /api/ue4ss/install-upstream`
- 保留本地 ZIP 安装入口
- `GET /api/workshop/mods`
- `POST /api/workshop/rescan`
- `POST /api/workshop/<id>/enable`
- `POST /api/workshop/<id>/disable`
- `GET /api/credits`
- `POST /api/system/open-trusted-link`

所有写操作使用现有 session cookie、CSRF/session token 边界、游戏级互斥锁和结构化错误。Workshop ID 只接受正十进制字符串；客户端不能提供文件路径、PackageName 或 URL 来绕过服务端扫描结果。

## 10. 测试策略

全程采用 TDD。

### 10.1 UE4SS

- 内置资源 manifest、大小、固定 SHA 和 ZIP 必需条目契约。
- frozen/dev 资源定位。
- 摘要或大小不匹配时拒绝安装。
- 固定 GitHub 仓库/Tag/资产名、无任意 URL、重定向主机限制、下载大小上限。
- GitHub digest 相同/不同/缺失/格式错误分支。
- Workshop UE4SS 冲突、游戏运行、目标不可写、事务回滚。
- LICENSE/NOTICE 随源码和构建产物存在。

### 10.2 Workshop

- 多 Steam Library fixture，但只访问 Palworld 固定路径。
- 严格 KeyValues 与 ACF 解析。
- `Info.json` 各 InstallRule、损坏字段、路径逃逸、reparse/junction。
- 缓存失效与手动重扫。
- INI 编码/BOM/换行/注释/未知键保持。
- 重复 ActiveModList 去重、启用/禁用幂等、全局开关。
- 依赖自动启用、缺失依赖、依赖环、禁用反向依赖确认。
- 游戏运行时拒绝写入。
- Workshop UE4SS 与手动 UE4SS 互斥。
- API 不接受任意路径、PackageName 或 URL。

### 10.3 樱花与 UI

- appearance schema 与旧配置迁移。
- 三种风格和四种密度的前后端契约。
- Canvas 生命周期、隐藏/恢复、reduced motion、resize、destroy。
- 固定随机数和模拟时钟下验证粒子边界与数量，不做脆弱的逐像素截图断言。
- 五个视图、按钮 action、键盘操作和 aria 状态契约。

### 10.4 发布验证

- 完整 pytest、所有前端 JS `node --check`、compileall、`git diff --check`。
- 两次全新 Python 3.13 hash-lock PyInstaller 构建，EXE/ZIP 哈希一致。
- ZIP CRC、sidecar 和第三方资源 SHA 检查。
- 正式 frozen EXE 烟测扩展到五视图、三樱花风格、内置 UE4SS 元数据和 Workshop 空/fixture 状态。

## 11. 发布与版本约束

- `APP_VERSION` 更新为 `2.1.0`。
- 构建脚本 packaged source paths 包含第三方 UE4SS 资源、credits 元数据及新增 Workshop 代码。

## 12. 验收标准

1. 当前塑料感椭圆樱花代码不存在，用户可选择三种全新风格并调节密度。

5. 扫描范围只覆盖 Steam 库中的 Palworld ACF/目录，不遍历其他游戏。
6. Workshop Mod 在统一列表中可识别、筛选并通过 ActiveModList 真实启停。
7. 启停不移动、不删除、不退订 Workshop 文件，并保留 PalModSettings.ini 非目标内容。
8. 开源致谢为独立页面，核心项目与完整直接依赖均有作者、许可证和可信链接。
9. 全部测试、两次可复现构建、真实 EXE 烟测、在线契约和独立审查通过。
10. `ylty1516/PalDeck` 存在已发布并重新下载校验成功的 `v2.1.0` Release。
