# PalDeck v2.3.1 Mod 数值调整设计

## 1. 目标

为能够可靠声明数值配置的受管 UE4SS Mod 增加安全检测和原位数值编辑。可调整的 Mod 条目右下角显示“调整数值 ▼”，点击后在条目内展开编辑器。PalDeck 不猜测任意 Lua 代码中的数字，也不修改 PAK 二进制或 Steam Workshop 缓存。

目标版本为 `2.3.1`。

## 2. 范围与边界

### 2.1 支持范围

仅支持当前 `ModService` 清单中的 `ModKind.UE4SS`：

- 启用和禁用状态：UE4SS Mod 文件始终位于 Mod 安装目录，状态只由相邻 `mods.txt` 控制。
- PalDeck 安装和外部发现的 UE4SS Mod 均可检测。

候选文件固定为 Mod 根目录的：

- `config.json`
- 可选 `palmod_config.json`

文件名匹配大小写不敏感，但同一规范名称出现多个文件时 fail-closed。

### 2.2 明确不支持

- PAK、LogicMods PAK 和 Steam Workshop Mod。
- `main.lua`、`config.lua` 或其他 Lua 脚本中的数字。
- 嵌套 JSON 数值、数组元素、布尔值、字符串或 `null`。
- 前端提供的本地路径、配置文件名、类型、范围或 schema。
- reparse point、符号链接、ADS、路径穿越、Windows 保留名和超大配置。

## 3. 配置识别

### 3.1 通用 `config.json`

`config.json` 必须：

- 是 UTF-8 普通文件，最大 256 KiB。
- 顶层是 JSON object。
- 最多 64 个顶层字段。
- 至少包含一个有限 `int` 或 `float`，布尔值不得视为整数。
- 可编辑 key 必须匹配 `[A-Za-z_][A-Za-z0-9_.-]{0,63}`。

没有描述文件时，只公开顶层有限数值字段：

- 标签为原始 key。
- 整数保持整数类型；浮点数保持浮点类型。
- 安全绝对范围为 `-1_000_000_000` 到 `1_000_000_000`。
- 不声明业务步长和说明。

### 3.2 `palmod_config.json` schema v1

描述文件必须是 UTF-8 普通文件，最大 256 KiB，且严格包含：

```json
{
  "schema_version": 1,
  "mod_id": "ConfigurableChestSlots",
  "display_name": "普通箱子格数调整",
  "description": "...",
  "version": "1.1.0",
  "fields": []
}
```

顶层不接受未知字段。`mod_id` 必须与 Mod 目录名称大小写不敏感相等。

每个 field 只允许：

- 必需：`key`、`label`、`type`、`default`。
- 可选：`min`、`max`、`step`、`description`。
- `type` 允许 `int`、`float` 或 `bool`；v2.3.1 严格验证但不在“调整数值”编辑器公开 bool 字段。
- key 必须唯一并存在于 `config.json`，且配置值类型匹配。
- 数值字段必须满足 `min <= default <= max`，当前值也必须在范围内。
- 数值字段的 `step` 必须是有限正数。
- int 字段的 min/max/default/step/current 均必须为整数。
- bool 字段的 default/current 必须是严格布尔值，且不得声明 min/max/step。
- 单个描述文件最多 32 个字段，至少包含一个 int 或 float 字段才显示调整入口。

描述文件损坏时整个 Mod 不显示调整入口，不降级为通用解析，避免绕过作者声明的约束。

## 4. 后端组件

新增 `backend/mod_value_service.py`，职责单一：

- `inspect_manifest(manifest) -> ModValueCapability | None`
- `read_values(manifest_id) -> dict`
- `update_values(manifest_id, values, revision) -> dict`

`ModService` 负责提供清单、统一游戏写锁、游戏运行检查和清单持久化；`ModValueService` 只通过服务端清单派生文件。

### 4.1 能力模型

列表接口为本地 Mod 增加派生字段：

```json
{
  "adjustable_values": true,
  "adjustable_value_count": 3
}
```

检测失败、格式不支持或路径不安全时返回 `false/0`，只读 Mod 列表不得因单个配置损坏而失败。

### 4.2 详情模型

`GET /api/mods/{mod_id}/values` 返回：

```json
{
  "mod_id": "...",
  "display_name": "普通箱子格数调整",
  "description": "...",
  "revision": "sha256:...",
  "fields": [
    {
      "key": "chest_slots",
      "label": "箱子目标格数",
      "type": "int",
      "value": 50,
      "min": 10,
      "max": 500,
      "step": 1,
      "description": "..."
    }
  ]
}
```

不返回配置绝对路径或未公开字段值。

### 4.3 更新请求

`POST /api/mods/{mod_id}/values` 只接受：

```json
{
  "revision": "sha256:...",
  "values": {"chest_slots": 60}
}
```

必须至少提交一个字段；不接受未知字段和额外顶层字段。revision 是读取时 `config.json` 原始字节的 SHA-256，防止覆盖用户或其他程序的并发修改。

## 5. 安全写事务

保存顺序固定为：

1. 获取现有统一游戏写锁。
2. 确认 Palworld 已停止。
3. 严格读取清单并确认 kind 为 UE4SS。
4. 从清单 `install_root` 派生 UE4SS 配置文件位置；禁用 UE4SS 仍使用相同文件位置。
5. 验证配置文件属于清单、路径无 reparse、是普通文件且大小合规。
6. 重新解析 schema 和当前配置。
7. 常量时间比较 revision。
8. 验证字段白名单、类型、有限性、范围和步长。
9. 复制当前 JSON object，只替换提交字段。
10. 使用 UTF-8、两空格缩进和结尾换行序列化；不改未提交的键和值。
11. 在配置所在卷创建临时文件，flush、`fsync` 后 `os.replace`。
12. 用新大小和 SHA-256 更新清单中对应的 `ManifestFile`，原子保存清单。
13. 清单保存失败时，将原始配置字节原子恢复；回滚失败升级为明确事务错误，不宣称成功。

配置之外的受管文件若处于 `missing` 或 `conflict` 状态则拒绝写入。配置自身与清单摘要不一致可以读取，但 revision 必须匹配当前字节；成功保存后以新摘要更新清单，承认用户当前配置状态。

## 6. API 与错误码

新增：

```text
GET  /api/mods/{mod_id}/values
POST /api/mods/{mod_id}/values
```

沿用 Cookie 会话和 `{ok,data}` 包络。错误码：

- `mod_values_not_supported`：Mod 或配置不支持。
- `mod_values_invalid`：schema、配置或提交值无效。
- `mod_values_stale`：revision 已变化。
- `mod_values_conflict`：非配置文件缺失/冲突或路径不安全。
- 现有 `game_running`、`permission_denied`、`not_found`。

响应不得包含本地绝对配置路径。

## 7. 前端交互

### 7.1 Mod 条目

只有 `adjustable_values === true` 时，在本地 Mod 条目右下角显示：

```text
调整数值 ▼
```

按钮通过现有动态事件委托处理，带 Mod ID，不携带路径。展开后变为“调整数值 ▲”，同一时间只展开一个 Mod。

### 7.2 内联编辑器

编辑器显示在当前 Mod 条目底部并横跨条目宽度：

- 标题、描述。
- 字段标签和数字 input。
- 作者声明的范围、步长和说明。
- “取消”和“保存并应用”。
- 加载、保存、成功和错误状态使用 `aria-live`。

取消不发请求。保存成功后用后端返回值刷新编辑器和 Mod 列表；stale 错误不覆盖输入，提示重新加载当前值。

`1236×771` 和 `960×640` 下字段改为单列，按钮换行且不得越界；键盘焦点顺序与视觉顺序一致。

## 8. 迁移与兼容

- 不修改现有 manifest schema；只在配置成功保存后更新已有 ManifestFile 摘要。
- 没有可识别配置的 Mod 行为和界面完全不变。
- 回收站、取消管理、UE4SS 框架生命周期、Workshop、Nexus 和更新功能保持现有边界。
- 版本升级到 `2.3.1`，发布 ZIP 继续使用确定性构建和 SHA-256 sidecar。

## 9. 测试与发布

### 后端

- 严格 schema、通用 JSON、int/float、范围、步长和字段数量。
- bool/NaN/Infinity/嵌套值/未知字段拒绝。
- 启用和禁用 UE4SS 状态下使用同一安全配置路径。
- PAK、LogicPak、Workshop 和无配置 Mod 不支持。
- stale revision、游戏运行、路径穿越、ADS、reparse、超大文件。
- 原子写入、清单更新失败回滚、并发游戏锁。

### API/前端

- 认证、请求字段白名单和错误映射。
- 能力标记、右下角“调整数值 ▼”、展开/收起和保存。
- DOM 文本安全、ARIA、重复动作防护。
- `1236×771`、`960×640` 无越界。
