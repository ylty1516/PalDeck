# 幻兽帕鲁 Mod 管理器重构设计

- 日期：2026-07-11
- 项目：`F:\Grok_Workspace\01_Projects\palworld-mod-manager`
- 目标平台：Windows 10/11、Steam 版 Palworld 客户端
- 发布形式：绿色便携 ZIP
- 技术路线：保留 Python、Flask、pywebview 与原生 HTML/CSS/JavaScript

## 1. 目标

在现有仓库基础上完成一个可交付的 Windows Mod 管理器，而不是仅修改视觉样式。产品必须做到：

1. 自动检测 Steam 版 Palworld 游戏目录，并允许手动选择。
2. 创建和识别 PAK、LogicMods、UE4SS 所需目录。
3. 导入 ZIP 或 PAK，识别 Mod 类型并安装到正确位置。
4. 为受管理 Mod 提供经过磁盘验证的启用、禁用和删除操作。
5. 只读连接 Nexus Mods，展示热门 Mod 的介绍、图片、作者、版本、统计和 N 网尾号。
6. 提供三套可切换主题、自定义背景和动态樱花效果。
7. 所有可见按钮必须绑定真实行为；失败时恢复 UI 状态并显示真实错误。
8. 提供自动化测试、构建脚本、便携产物和验证记录。

## 2. 已确认范围

### 2.1 支持

- Windows 10/11。
- Steam 客户端版 Palworld。
- 普通 PAK Mod。
- LogicMods。
- UE4SS Lua/脚本 Mod。
- ZIP 和直接 PAK 导入。
- 同名 `.pak`、`.utoc`、`.ucas` 文件组。
- Nexus Mods 匿名只读目录与搜索。
- 便携数据目录。

### 2.2 不支持

- Xbox / Microsoft Store 版 Palworld。
- Palworld 专用服务器 Mod 管理。
- Steam Workshop 订阅、取消订阅或缓存修改。
- Nexus 登录、下载或自动安装。
- 网页抓取作为 Nexus API 的替代方案。
- 在公开程序中内置开发者个人 Nexus API Key。

不支持项不得以不可工作的按钮或模拟状态出现在 UI 中。

## 3. 总体架构

```text
launcher.py / pywebview
  └─ frontend
      ├─ 我的模组
      ├─ 导入安装
      ├─ N 网热门
      └─ 设置与外观
          ↓ 带会话令牌的本机 API
backend/app.py
  ├─ GameLocator      游戏检测、路径验证、进程与权限检查
  ├─ ModInstaller     安全解包、类型识别、事务安装
  ├─ ModState         扫描、启停、删除、清单校验
  ├─ Appearance       主题、背景导入和设置
  └─ NexusCatalog     匿名只读查询、规范化、短缓存与降级
          ↓
Palworld 游戏目录 / 程序旁 data 目录 / Nexus GraphQL
```

Flask 仅监听 `127.0.0.1`，启动时选择可用随机端口，并生成进程级会话令牌。前端 API 请求必须携带令牌；静态资源和健康检查以最小范围开放。

模块通过清晰接口通信，文件系统操作不得直接散落在路由函数中。

## 4. 便携目录与数据模型

发布目录结构：

```text
PalDeck/
  PalDeck.exe
  data/
    config.json
    manifests/
      <mod-id>.json
    backgrounds/
    disabled/
      <mod-id>/
    cache/
      nexus/
    logs/
```

`config.json` 保存游戏路径、主题、背景、遮罩、模糊度、背景位置、樱花密度和窗口设置。

每个 Mod 清单至少保存：

- 稳定 UUID。
- 显示名称和 Mod 类型。
- 来源文件名。
- 安装时间。
- 启用状态。
- Nexus 尾号（若用户填写或可识别）。
- 每个归属文件的相对路径、大小和 SHA-256。
- 安装目标根目录。
- UE4SS 原始 `enabled.txt` 状态。
- 外部修改、缺失或冲突状态。

配置和清单采用临时文件写入后原子替换，避免进程中断造成半写 JSON。

## 5. 游戏检测与目录

### 5.1 自动检测

检测顺序：

1. Steam 注册表安装路径。
2. `steamapps/libraryfolders.vdf` 中的所有库。
3. 各库的 `steamapps/appmanifest_1623730.acf`。
4. 清单中的 `installdir` 与实际游戏目录交叉验证。
5. 常见路径仅作为最后降级，不限制盘符范围。

有效游戏根必须包含 `Palworld.exe`，或同时存在预期 Win64 可执行文件和 `Pal/Content/Paks`。仅存在一个 `Paks` 目录不足以判定有效。

手动选择路径时使用相同验证规则。

### 5.2 Mod 目录

- PAK：`Pal/Content/Paks/~mods`
- LogicMods：`Pal/Content/Paks/LogicMods`
- UE4SS：探测 `Pal/Binaries/Win64/Mods` 或 `Pal/Binaries/Win64/ue4ss/Mods`

UE4SS 根目录通过 `mods.txt`、`UE4SS-settings.ini`、UE4SS DLL 和已有目录共同判断，不依赖单一固定路径。

状态查询不得隐式创建目录或修改配置。只有用户保存游戏路径、点击修复目录或开始安装时才允许写入。

## 6. Mod 导入与安装

### 6.1 输入与安全

支持：

- `.zip`
- `.pak`
- 直接 PAK 导入时，自动收集同目录同基名的 `.utoc` 和 `.ucas`

ZIP 解包必须阻止：

- 绝对路径和 `..` 路径穿越。
- Windows 盘符逃逸。
- 符号链接和特殊文件。
- 超过配置上限的文件数量、单文件大小和总展开大小。
- 损坏或加密压缩包。

所有检测先在程序数据目录的暂存区完成。

### 6.2 类型适配器

#### PAK

默认安装到 `~mods`。同名 `.pak/.utoc/.ucas` 作为不可拆分文件组。压缩包中的多组 PAK 分别建立清单，或在明确属于同一发布包时建立一个包含多个文件组的清单。

#### LogicMods

当压缩包目录语义包含 `LogicMods`，或用户明确选择 LogicMods 时，安装到 `Pal/Content/Paks/LogicMods`。不得仅凭任意 PAK 文件猜为 LogicMods。

#### UE4SS

识别 `Scripts/main.lua` 或标准 UE4SS Mod 目录。安装到探测出的真实 UE4SS Mods 根目录。受管理 Mod 统一由 `mods.txt` 控制；包内 `enabled.txt` 被安全收纳并记录，避免其绕过开关。

### 6.3 事务与冲突

安装流程：

1. 确认游戏未运行。
2. 验证输入并计算哈希。
3. 检测类型和目标。
4. 检查同名路径、现有清单和哈希冲突。
5. 若有冲突，返回明确冲突信息，由用户选择覆盖、保留两份或取消。
6. 将文件复制到目标旁暂存位置。
7. 原子移动到最终位置。
8. 原子写入清单。
9. 重新扫描并验证磁盘状态。
10. 任一步失败则回滚本次新建文件和配置改动。

不得以随机改名静默掩盖冲突。

## 7. 启用、禁用、删除与同步

### 7.1 PAK 与 LogicMods

禁用时将清单中的整个文件组原子移动到 `data/disabled/<mod-id>`。启用时先检查目标冲突，再原子恢复。操作后重新计算实际状态。

不使用仅修改 `.pak.disabled` 扩展名的方式，因为同组 `.utoc/.ucas` 仍可能残留或造成状态不一致。

### 7.2 UE4SS

- 启用：在真实 `mods.txt` 中写入 `<ModName> : 1`。
- 禁用：写入 `<ModName> : 0`。
- 安装时收纳包内 `enabled.txt`，防止绕过 `mods.txt`。
- 更新 `mods.txt` 时保留未知行、注释和其他 Mod 顺序，不整体重写用户文件。

### 7.3 删除

删除仅处理清单归属文件和由本工具创建且已为空的目录。若文件哈希与清单不同，UI 必须警告外部修改，并要求用户再次确认。

### 7.4 重新扫描

扫描现有 PAK/LogicMods/UE4SS Mod 并建立可管理清单。PAK 文件按同基名分组。已存在清单以真实磁盘状态为准，并标记：

- 正常。
- 已禁用。
- 文件缺失。
- 外部修改。
- 目标冲突。

界面不得仅使用上次保存的布尔值宣称状态。

## 8. 外观与交互

### 8.1 默认背景

用户提供的源图片：

`C:\Users\yyx\Pictures\Screenshots\屏幕截图 2026-07-11 165046.png`

构建时复制到 F 盘项目资源，并裁掉顶部监控文字。最终程序不得依赖该 C 盘源路径。发布前应由项目所有者确认拥有该图片的再分发权。

### 8.2 三套主题

- 星空极光玻璃：浅色磨砂内容区、深蓝侧栏、蓝粉渐变强调。
- 象牙白樱花：高亮度象牙白面板、粉色强调、较弱背景存在感。
- 深空夜幕：深蓝半透明面板、蓝紫霓虹强调。

三套主题共享同一 DOM 和组件，通过 CSS 变量切换，不复制三份页面。用户选择即时生效并持久保存。

### 8.3 背景设置

支持 PNG、JPG、JPEG 和 WEBP。后端验证 MIME、实际图像格式、大小和尺寸后，将图片复制到 `data/backgrounds`。提供：

- 更换背景。
- 恢复默认。
- 背景遮罩强度。
- 背景模糊度。
- 背景显示位置。
- 预览与保存。

删除旧背景时只删除本工具复制到数据目录的文件，不触碰用户原始图片。

### 8.4 樱花与按钮

樱花使用单一 Canvas 和 `requestAnimationFrame`，`pointer-events: none`。提供关闭、低、中、高密度。页面隐藏或窗口不可见时暂停；系统要求减少动态效果时默认关闭。

按钮统一提供：

- Hover 状态。
- 按压位移和缩放。
- 涟漪反馈。
- 键盘焦点。
- 加载中状态。
- 禁用状态。

异步操作期间防止重复点击。操作失败时控件恢复原状态。

## 9. 页面设计

### 9.1 我的模组

显示名称、类型、实际状态、大小、Nexus 尾号、路径、文件完整性和备注。提供筛选、刷新、启停、打开目录和删除。

### 9.2 导入安装

支持拖放和文件选择；展示自动检测结果、目标目录、安全警告和安装进度。冲突时显示明确决策对话框。

### 9.3 N 网热门

提供：

- 总下载热门。
- 推荐热门。
- 最新上传。
- 名称搜索。
- 纯数字尾号查询。

卡片展示介绍图、简介、作者、版本、下载量、推荐量和尾号。只有“打开 N 网”和“复制尾号”操作，不提供下载按钮。

### 9.4 设置与外观

包含游戏路径检测、目录修复、UE4SS 状态、主题、背景、樱花和程序更新设置。需要写权限但当前不足时提供以管理员身份重启入口。

## 10. Nexus 数据策略

使用 Nexus V2 GraphQL 匿名只读查询，不内置 API Key，不抓取网页。每次进入页面或用户主动刷新时请求实时数据。

- 连接和读取设置明确超时。
- 对 GraphQL 错误和字段变化返回可读错误。
- 对响应做严格规范化，缺失可选字段时使用安全默认值。
- 元数据使用 10 分钟短缓存减少重复请求。
- 实时请求失败时可展示最后一次成功缓存，但必须标注缓存时间和离线状态。
- 图片使用 Nexus 返回的公开图片 URL；加载失败显示本地占位图。
- 查询参数、返回 URL 和渲染内容均做转义和验证。

“热门”语义必须在 UI 中明确为总下载量或推荐量排序，不将其伪装为 Nexus 官方趋势榜。

## 11. 错误与权限处理

所有写操作前检查 Palworld 进程。游戏运行时拒绝安装、启停和删除，并提示用户退出游戏。

检查游戏目录写权限；不足时提供“以管理员身份重启”。重启前保存安全的待恢复 UI 状态，但不保存上传文件路径或敏感数据。

错误分类至少包括：

- 游戏目录无效。
- 游戏正在运行。
- 无写权限。
- ZIP 损坏、加密或不安全。
- 不支持的 Mod 类型。
- 文件冲突。
- 磁盘空间不足。
- 文件被占用。
- 回滚失败。
- Nexus 超时、限流或响应变化。

后端日志不得记录 Nexus 凭据、用户私密路径以外的无关信息或完整异常页面。前端只展示经过整理的错误消息。
