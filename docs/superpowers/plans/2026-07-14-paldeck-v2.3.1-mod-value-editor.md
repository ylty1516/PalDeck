# PalDeck v2.3.1 Mod 数值调整实现计划

> **面向 AI 代理的工作者：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法来跟踪进度。

**目标：** 安全识别受管 UE4SS Mod 的 JSON 数值配置，在 Mod 条目右下角提供“调整数值 ▼”内联编辑器，并发布 v2.3.1 便携包。

**架构：** 新增独立 `ModValueService`，只从后端清单派生 `config.json` 和可选 schema，负责严格解析、revision 和事务更新。`ModService` 组合该服务并向列表派生能力字段；Flask 提供固定 GET/POST API；前端继续使用动态事件委托并在单个 Mod 行中展开编辑器。

**技术栈：** Python 3.13、Flask、pathlib/JSON/SHA-256、原生 ES Modules/CSS、pytest、Node test、PyInstaller。

定义：

```python
@dataclass(frozen=True)
class NumericField:
    key: str
    label: str
    kind: Literal["int", "float"]
    value: int | float
    minimum: int | float
    maximum: int | float
    step: int | float | None
    description: str

@dataclass(frozen=True)
class ModValueCapability:
    display_name: str
    description: str
    revision: str
    fields: tuple[NumericField, ...]
    config_relative_path: str
```

候选文件只从 manifest files 中大小写唯一匹配根级 `config.json` 和 `palmod_config.json`；读取时从 UE4SS `install_root` 重新派生文件，限制 256 KiB，拒绝 reparse，JSON 常量 `NaN/Infinity` 通过 `parse_constant` 抛错。
