# PalDeck v2.3.4 发布验证

## 本次范围

- 普通受管 Mod 逐文件健康报告：分别记录启用区与禁用区的正常、缺失、修改和冲突状态。
- 安装时创建 SHA-256 内容寻址健康副本，相同内容只保存一份。
- 缺失文件可从已验证健康副本无确认恢复。
- 修改文件必须确认；执行时先移入 `data/repair-quarantine`，再恢复健康副本。
- 修复计划带磁盘状态 revision，文件变化后旧计划拒绝执行。
- Mod 清单在事务提交后保存最后正常备份；损坏清单可由健康中心安全恢复，原损坏文件保留在隔离区。
- 设置页增加 Mod 健康中心；异常 Mod 卡片增加逐文件摘要和“诊断修复”。
- “重扫”不再作为异常 Mod 的修复动作，避免把重新发现与内容恢复混淆。

## 安全边界

- 游戏运行时拒绝所有修复写入。
- 缺少可信健康副本时不猜测、不覆盖，只报告 `trusted_source_missing`。
- 重解析点、目录占位和无法安全解析的路径保持 fail-closed。
- 修改文件修复前必须显式确认，原内容进入唯一事务隔离目录。
- 修复发布后重新执行清单审计；异常未完全解决时返回部分完成，不虚报成功。
- 清单备份只在上层事务完成后 checkpoint，失败事务不会污染最后正常备份。
