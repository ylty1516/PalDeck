# PalDeck v2.3.5 发布验证

## 本次范围

- 新增“游戏故障恢复”设置卡片，后台每 5 秒保守观察 Palworld 是否运行。
- 游戏从运行变为退出后不猜测是否崩溃，由用户明确选择“本次运行正常”或“游戏出现故障”。
- 首次启用该功能时按游戏路径建立稳定基线；正常会话确认后，把该次实际运行前的 Mod 快照提升为新的稳定基线。
- 故障会话只把相对稳定基线中新启用的本地受管 Mod 和 Steam Workshop Mod 列为自动回滚候选。
- 回滚只执行禁用，不删除、不卸载、不恢复未知内容；被禁用或缺失的旧 Mod 不会被自动重新启用。
- 多 Mod 回滚使用共享跨进程锁；中途失败会按逆序恢复本次已经禁用的 Mod，并保留原恢复计划以便重新诊断。

## 安全边界

- 游戏运行期间只能记录会话和读取状态；会话评估与回滚写入均返回 `423 game_running`。
- 回滚必须先把会话标记为故障，再提交显式 `confirm: true` 和格式严格的 SHA-256 revision。
- revision 覆盖会话、稳定基线、当前 Mod 状态、动作和阻塞项；状态改变后旧计划返回 `recovery_plan_stale`。
- 文件状态异常或不可安全启停的 Mod 仅报告 `state_not_toggleable`，不强制移动文件。
- 状态文件按游戏路径隔离；损坏、字段缺失、重复或伪造的稳定快照不会作为回滚依据，而是安全重建当前基线。
- 该功能不读取游戏崩溃日志，也不宣称能判断运行时兼容性；它只根据用户确认和可验证的启停差异提供回滚。
